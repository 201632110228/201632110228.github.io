CSS ID selector

In some respects, the ID selector is similar to the class selector, but there are some important differences.

First, there is a hash number in front of the ID selector, which is also called a board number or a well number.

Check out the following rules:

*#intro {font-weight:bold;}

Like the class selector, you can ignore the wildcard selector in the ID selector. The previous examples can also be written:

#intro {font-weight:bold;}

The effect of this selector will be the same

The second difference is that the ID selector does not reference the value of the class attribute, which, without a doubt, refers to the value in the ID attribute.

Here's an example of an actual ID selector:

<p id="intro">This is a paragraph of introduction.</p>

    • Class selector or ID selector?

    In the chapter on class selectors, we've explained that you can specify classes for any number of elements.The class name important in the previous chapter is applied to the p and h1 elements, and it can be applied to more elements.

    Difference 1: only once in the document

    Unlike classes, in an HTML document, the ID selector is used once, and only once.

    Difference 2: you cannot use an ID word list

    Unlike the class selector, the ID selector cannot be used in combination because the ID attribute

    Difference 3: ID can have more meaning

    Similar to classes, you can select ids independently of the elements.In some cases, you know that a particular ID value appears in the document, but you don't know which element it will appear on, so you want to declare an independent ID selector.For example, you might know that a given document will have an element with an ID value of mostImportant.You don't know whether the most important thing is a paragraph, a phrase, a list item, or a section title.You only know that each document has such an important content that it can be in any element and only one.In this case, you can write the following rules:

    #mostImportant {color:red; background:yellow;}

    This rule matches the following elements (these elements cannot appear in the same document, because they all have the same ID value) :

    <h1 id="mostImportant">This is important!</h1>

    <em id="mostImportant">This is important!</em>

    <ul id="mostImportant">This is important!</ul>
